Traditional websites vs SPA

  1. Speed
  2. Code complexity

Code complexity

Traditional website

  • Fast first load
  • Reload of page on every call

SPA

  • Slow first load due big js bundle
  • No need for reload

Code complexity

Traditional website

  • Frontend is highly coupled with backend
  • Hard to maintain state

SPA

  • SPA is similar to mobile apps. It's completely separate from backend
  • Great developer experience

SEO and social media sharing

Traditional website

  • Works without js(pure html/css from server)
  • Good SEO support

SPA

  • Need js to work properly
  • It takes additional effort to make it work with SEO and social media sharing

Component lifecycle

Lets write Clock

Constructor

Is equal to

Constructor

  • Don't forget to call super(props);
  • Good for init of state
  • Bad for using this.setState()
  • Bad for running side effects
    • No api calls
    • No dom manipulations

Render

Render

  • It's only required method in a class component.
  • Called only on
    • New props from parent
    • this.setState()
    • this.forceUpdate()
  • Should be pure
    • No side effects
    • No dom manipulations

componentDidMount

componentDidMount

  • Called after first render
  • You can call this.setState() here
  • Should be used for running side effects
    • Fetching data from server
    • Subscriptions
    • Timers
    • Manipulations that require dom

componentWillUnmount

componentWillUnmount

  • Called before component will be removed from DOM and destroyed
  • You should not call this.setState() here
  • Should be used for cancelling
    • Async requests to server
    • Subscriptions
    • Timers
